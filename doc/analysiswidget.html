typeCase widgets: analysis

The analysis widget is the widget where the actual analysis takes place. It receives all selected parameters from the algorithm widget and the data-base widget.

The main part of the widget is covered by a canvas that displays during analysis information about the current analysis. This can be the event pattern (see picture) or a histogram, the leaf of a tree or other information

The colored button on the bottom shows the status of the analysis. red for non initialized, yellow for initalized, green for running.

The following buttons have this effects:

Algorithm
opens the OrderSelectionWidget. Here you can specify in which order the selected algorithms should be executed for each event. You can also add Data-Input- and -Output-Algorithms along with reaction-recognition algorithms.
Watch
opens the ViewSelectionWidget where you can specify which information about the running analysis you want to see during analysis.
Help
opens the helpWidget with information about the analysis widget. In essence: this page.
Init
Initializes the analysis with the current algorithm and data-basis parameter sets. Setup is initialized with the files provided by the first run.
Start
If initialized, this button starts the analysis which will run until stoped or ran out of events to analyse.
Step
If initialized, this button starts the analysis for a specified number of events (see input on bottom of widget) or until end of events whichever happens earlier.
Stop
If running this button stops the analysis.
Final
If initialized this button finalizes the analysis. This calls the destructors for all of the algorithms and is essential for the writing algorithms to close the open files and also for the algorithms that do the main part of the analysis during finalisation.
