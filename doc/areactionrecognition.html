This algorithm belongs to the basic algorithms. As the tofAnalysis the algorithm AReactionRecognition does not do any reaction-recognition itself, but it hosts the reaction classes that do the reaction-recognition themselves and only starts the RR and sorts the possible reactions for the event itself.

    • AReactionRecognition::AReactionRecognition(TSetup& setup, TEvent& eventIn, AFitAlgorithm* kinfitIn, vector<algorithm_parameter> processes)
      Constructor. The setup is needed to get the initial conditions (beam(s) and target) for the reaction recognition, eventIn is needed to get the event-information, like tracks, trigger, etc. The kinfit-variable is passed to the reactions if it is not NULL. Most likely it is not used, but the kinFit from the fit-package is used. In processes a description of the reactions to check including cuts, etc. is passed to the RR.
