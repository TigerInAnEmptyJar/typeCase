    • void functionNameTypeDoc::parse(const string& s)
      Parses a string representing a function in C++ code. Ignores comments, handles operators and inline definitions and finds the parameter list. The string right of the parameter-list is saved in the Limiter variable, the left is decomposed in specifier, type and name, where constructors and destructors are recogniced and handled differently. The parameter-list is parsed for its content.

    • string functionNameTypeDoc::htmlHREF() const
      Returns a string containing the html-link to the html-description of the function. If a documentation-file for the type is given or it is listed in the list of known types or classes, this link is included. The same is done for the parameters. Defaults are shown here. Constructor case is taken into account. Viewn with a browser, it looks like a C++-definition of a function.
      top
    • vector<string> functionNameTypeDoc::htmlNAME(const string& classname = "") const
      Returns the html-documentation of the function as a vector of strings. If the classname is specified, the function name is given by "classname::functionname". A html-"name"-label is included with the current Doc-string. If name and type are equal (constructor case), type is left out. If a documentation file for the type is specified or listed in the known types or classes, a link is included there, else the ype is colored gray. The parameters are given as in nameTypeDoc::htmlNAME(). The description is added in green.
